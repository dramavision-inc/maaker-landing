# Maaker.AI 落地页设计方案

## 目标

承接小红书引流，作为 Martin 的个人品牌官网。展示"我是谁、怎么联系我"，引导进群转化。

## 用户路径

```
小红书笔记 → 点击主页链接/搜索 MaakerAI → maaker.ai
  → 了解这个人是谁（建立信任）
  → 找到联系方式 / 加入社群（转化）
```

## 页面结构

### 1. Hero（首屏）
- 头像（圆形）
- 品牌名：Maaker.AI
- 一句话：一个人类老板 + 全AI员工的一人公司
- 副标题：独立开发者 · 全职 Vibe Coder

### 2. 关于我
- 2-3 段简短介绍，口语化
- 核心信息：辞职做独立开发、用 AI 当员工、做小而美的工具
- 手绘涂鸦装饰

### 3. 联系方式
- 小红书（主要平台）
- 微信号：MaakerAI
- 邮箱（备用）

### 4. 进群引导
- 移动端：固定底栏 CTA
- 桌面端：页面底部区块
- 话术：加入 AI 独立开发营，和同行一起交流

## 视觉设计

- **布局**：单列居中，max-width 640px，移动端优先
- **留白**：大量，呼吸感
- **配色**：白底 #FFFFFF，主文字 #111，副文字 #666，点缀 teal #0D9488
- **字体**：系统字体栈（-apple-system, sans-serif）
- **装饰**：手绘涂鸦风插图（SVG），黑线条 + teal 小面积点缀
- **卡片**：圆角 16px，柔和阴影 shadow-sm
- **动画**：轻微 fade-in 滚动出现，不花哨

## 技术方案

- 纯静态 HTML + Tailwind CSS（CDN）
- 无框架，无构建步骤
- 部署：Cloudflare Pages
- 域名：maaker.ai
- 手绘插图：SVG inline 或图片

## 文案方向

真实、口语化、不端着。参考小红书爆款笔记的风格——第一人称，像跟朋友聊天。
